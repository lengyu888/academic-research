# 科研报告模块

## 触发场景

- 用户需要写开题报告
- 用户需要写实验报告
- 用户需要写阶段性研究汇报
- 用户需要写文献调研报告
- 用户需要总结研究成果

## 工作流程

### 1. 理解报告需求

明确以下信息：
- **报告类型**：开题报告、实验报告、进展报告、结题报告
- **研究主题**：研究问题和领域
- **报告要求**：格式、字数、结构要求
- **已有材料**：用户已经准备了哪些内容

### 2. 报告类型与结构

#### 开题报告
```
1. 研究背景与意义
   - 研究背景
   - 研究意义（理论/实践）
   
2. 国内外研究现状
   - 国外研究现状
   - 国内研究现状
   - 文献评述
   
3. 研究内容与目标
   - 研究内容
   - 研究目标
   - 拟解决的关键问题
   
4. 研究方案与技术路线
   - 研究方法
   - 技术路线图
   - 可行性分析
   
5. 研究计划与进度安排
   
6. 参考文献
```

#### 实验报告
```
1. 实验目的
   
2. 实验原理
   - 理论基础
   - 实验假设
   
3. 实验设计
   - 实验材料/数据
   - 实验步骤
   - 变量控制
   
4. 实验结果
   - 数据记录
   - 结果展示
   - 统计分析
   
5. 讨论与分析
   - 结果解释
   - 误差分析
   - 与预期对比
   
6. 结论
   
7. 参考文献
```

#### 研究进展报告
```
1. 研究概述
   - 研究背景（简要）
   - 研究目标
   
2. 已完成工作
   - 文献调研
   - 方法设计
   - 实验进展
   
3. 阶段性成果
   - 主要发现
   - 数据/结果
   
4. 存在问题与困难
   
5. 下一步计划
   
6. 需要的支持/资源
```

### 3. 写作要点

#### 研究背景
- 从大背景切入，逐步聚焦到具体问题
- 说明为什么这个问题重要
- 引用相关数据或案例支撑

#### 研究现状
- 按主题组织，不要流水账
- 评述现有工作的优缺点
- 自然引出研究空白

#### 研究方案
- 方法选择要有依据
- 技术路线要清晰可执行
- 说明创新点和可行性

#### 结果展示
- 使用图表直观展示
- 数据要经过统计分析
- 结果要与目标对应

### 4. 常见问题

| 问题 | 表现 | 改进方向 |
|------|------|----------|
| 背景过长 | 大段无关背景 | 聚焦与研究直接相关的背景 |
| 现状罗列 | 缺少评述和总结 | 加入分析和研究空白 |
| 目标模糊 | "研究XX"过于笼统 | 分解为具体可验证的子目标 |
| 方案不可行 | 技术路线过于理想 | 考虑资源限制和风险 |
| 结果无分析 | 只列数据不解释 | 深入讨论结果的意义 |

## 注意事项

- 报告内容要基于用户提供的实际研究内容，不要编造数据或结果
- 如果用户只提供了部分信息，基于已有信息给出框架，并标注需要补充的部分
- 注意报告的格式要求（学校、学院可能有特定模板）
- 参考文献格式要统一（APA、IEEE、GB/T 7714 等）
- 语言要正式、客观、严谨
