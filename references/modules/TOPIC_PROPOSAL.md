# 选题开题模块

## 触发场景

- 用户不知道研究方向怎么选
- 用户需要评估课题可行性和创新性
- 用户需要撰写开题报告
- 用户需要在几个候选方向中做决策
- 用户需要提炼研究创新点

## 工作流程

### 1. 理解选题背景

明确以下信息：
- **研究领域**：用户所在的大方向（如计算机视觉、自然语言处理）
- **个人条件**：已有基础、可用资源、导师方向、时间周期
- **目标产出**：期刊/会议论文、学位论文、项目结题
- **候选方向**：如有，列出正在考虑的几个方向

### 2. 选题评估五维模型

| 维度 | 评估问题 | 高价值标志 |
|------|----------|-----------|
| 创新性 | 与现有工作的本质区别在哪？ | 首次解决/换角度切入/跨领域组合 |
| 可行性 | 给定资源能否完成？ | 数据可得、计算资源充足、时间在预算内 |
| 重要性 | 解决了什么实际或学术问题？ | 有明确的应用场景或理论基础价值 |
| 可扩展性 | 能否拆成多个子课题？ | 核心突破后可派生2-3篇后续工作 |
| 团队匹配 | 导师方向和你的技能是否匹配？ | 有积累、有竞争力、有兴趣 |

加权评估：
```
总分 = 创新性×0.3 + 可行性×0.3 + 重要性×0.2 + 可扩展性×0.1 + 团队匹配×0.1
```

### 3. 创新点提炼

#### 方法

从以下维度切入分析：

**问题创新**：研究前人没解决或解决得不好的问题
- 例：现有语义分割只做像素分类→ 首次做像素级关系推理

**方法创新**：用新方法解决已有问题
- 例：用 Transformer 替代 CNN 做图像分割 → ViT-Seg

**数据创新**：构建新数据集或发现新数据规律
- 例：首个中文多模态情感数据集

**应用创新**：将已有方法迁移到新场景
- 例：将目标检测用于工业缺陷检测

#### 创新点矩阵（帮助定位）

```
                方法新          方法现有
              ┌─────────────┬─────────────┐
问题新         │  理想位置     │  定义新问题   │
              ├─────────────┼─────────────┤
问题现有       │  改进型贡献   │  无效/低价值   │
              └─────────────┘─────────────┘
```

### 4. 开题报告结构

```
1. 选题背景与意义（1-2页）
   - 研究背景 → 问题提出 → 研究意义

2. 国内外研究现状（3-5页）
   - 按主题组织，突出研究空白
   - 可用 literature-review 模块生成

3. 研究目标与内容（2-3页）
   - 总目标 → 分解为 3-4 个子目标
   - 每个子目标对应的研究内容

4. 技术路线与方案（2-3页）
   - 技术框架图
   - 关键方法描述
   - 可行性分析

5. 创新点与预期成果（1页）
   - 3-4个创新点
   - 预期发表论文数、专利数

6. 研究计划与进度安排（1页）
   - 分阶段时间表
   - 里程碑节点

7. 参考文献
```

### 5. 选题决策建议

#### 避免的选题陷阱

| 陷阱 | 表现 | 规避方法 |
|------|------|----------|
| 题目过大 | "基于深度学习的图像理解" | 限定范围：任务、场景、方法 |
| 题目过窄 | 一个小 trick 改进 | 评估 trick 的泛化性和学术价值 |
| 热门跟风 | SOTA 刷榜但无实质创新 | 找冷门但有价值的切入点 |
| 重复前人 | 文献调研不充分 | 用 literature-review 模块做系统综述 |
| 脱离实际 | 没有可用数据或计算资源 | 先做预实验验证可行性 |

## 输出模板

```markdown
## 选题评估报告

### 1. 候选方向评估
| 方向 | 创新性 | 可行性 | 重要性 | 可扩展性 | 匹配度 | 综合评分 |
|------|--------|--------|--------|----------|--------|----------|
| [方向1] | ★★★★ | ★★★ | ... | | | 3.6/5 |
| [方向2] | ★★★ | ★★★★★ | ... | | | 3.8/5 |

### 2. 推荐方向：[方向名]
**核心研究问题**：[一句话]
**与众不同的地方**：[2-3句]
**预期产出**：[论文数/专利数/系统]
**风险评估**：[主要风险 + 应对方案]

### 3. 创新点矩阵
1. [创新点1名称]：[一句话描述] → [问题/方法/数据/应用]创新
2. [创新点2名称]：[一句话描述]

### 4. 下一步行动
1. 立即：[最关键的前期准备]
2. 第一周：[计划]
3. 第一个月：[里程碑]
```

## 注意事项

- 选题是研究中最关键的决策，花足够时间调研和讨论
- 好题目的标准：一句话能说清楚、有差异性、有可行性
- 导师的建议权重最高——了解领域动态和评审偏好
- 不要追求"颠覆性"，好的增量创新也能发好论文
- 开题报告可以调用 `literature-review` 模块生成研究现状部分

## 输出格式

默认 Markdown。用户要求 DOCX 或 LaTeX 时，参考 `OUTPUT_DOCX.md` 或 `OUTPUT_LATEX.md`。
