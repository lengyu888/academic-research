# 实验设计模块

## 触发场景

- 用户需要设计实验方案
- 用户需要估算样本量
- 用户需要确定变量控制策略
- 用户需要评估实验方案的可行性
- 用户需要对现有实验方案做改进

## 工作流程

### 1. 理解实验需求

明确以下信息：
- **研究问题**：要验证什么假设
- **实验类型**：用户实验、系统性能实验、算法对比实验、问卷调查
- **变量**：自变量（操纵）、因变量（测量）、控制变量（保持恒定）
- **约束**：被试数量、时间、设备、预算

### 2. 实验设计类型选择

| 实验类型 | 适用场景 | 典型设计 |
|----------|----------|----------|
| 算法对比实验 | 模型性能比较 | 多组独立设计 + 重复测量 |
| 用户实验 | 人机交互、可用性 | 被试内/被试间/混合设计 |
| 问卷调查 | 大规模数据采集 | Likert量表 + 验证性因素 |
| 消融实验 | 组件贡献度 | 控制变量法，每次只变一个组件 |
| 鲁棒性验证 | 模型稳定性 | 交叉验证 + 扰动测试 |
| A/B 测试 | 在线效果对比 | 随机分组 + 统计检验 |

### 3. 样本量估算

```python
import numpy as np
from scipy import stats

def sample_size_power(effect_size=0.5, alpha=0.05, power=0.80, design='two-sample'):
    """基于效应量和期望统计功效估算样本量
    
    Cohen's d 效应量：
      0.2 小效应 → 需要大样本 (每组 ~394)
      0.5 中效应 → 需要中样本 (每组 ~64)
      0.8 大效应 → 需要小样本 (每组 ~26)
    """
    if design == 'two-sample':
        z_alpha = stats.norm.ppf(1 - alpha / 2)
        z_beta = stats.norm.ppf(power)
        n_per_group = 2 * ((z_alpha + z_beta) / effect_size) ** 2
        return int(np.ceil(n_per_group))
    elif design == 'paired':
        z_alpha = stats.norm.ppf(1 - alpha / 2)
        z_beta = stats.norm.ppf(power)
        n = ((z_alpha + z_beta) / effect_size) ** 2
        return int(np.ceil(n))

# 例子：检测中等效应量
n = sample_size_power(effect_size=0.5, design='two-sample')
print(f"每组需要 {n} 个被试 (d=0.5, alpha=0.05, power=0.80)")
# → 每组需要 64 个被试
```

### 4. 变量控制清单

- [ ] **自变量**：操纵水平是否充分？是否有对照组/基线？
- [ ] **因变量**：测量指标是否可靠（信度）且有效（效度）？
- [ ] **混淆变量**：哪些未控制变量可能影响结果？（列出并说明控制方式）
- [ ] **随机化**：被试/样本是否随机分配到各组？
- [ ] **平衡**：顺序效应（拉丁方）、疲劳效应（休息间隔）
- [ ] **盲法**：是否单盲/双盲？（用户实验适用）

### 5. 实验方案评估

| 评估维度 | 检查问题 |
|----------|----------|
| 内部效度 | 因变量的变化确实由自变量引起？ |
| 外部效度 | 结论能否推广到其他人群/场景？ |
| 统计功效 | 样本量是否足够检测预期效应？ |
| 可复现性 | 他人能否按照方案得到相同结果？ |
| 可行性 | 在给定资源约束下能否完成？ |

## 注意事项

- 实验设计在数据采集前完成，不要在分析阶段才想控制变量
- 样本量不在大小，而在于能否检测到实际有意义的效应
- 消融实验每次只变一个组件，否则无法归因
- 实验方案应在注册或预注册后再执行（如适用）
