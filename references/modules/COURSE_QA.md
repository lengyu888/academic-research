# 课程答疑模块

## 触发场景

- 用户对课程概念有疑问
- 用户需要解释某个知识点
- 用户需要作业辅导（不是代做）
- 用户想梳理某个主题的知识体系

## 工作流程

### 1. 理解问题

明确以下信息：
- **课程/领域**：数学、计算机、物理、经济学等
- **具体问题**：概念解释、计算方法、证明思路
- **学习阶段**：本科、研究生
- **已有理解**：用户已经知道什么，在哪里卡住

### 2. 答疑原则

- **启发式教学**：引导思考，不直接给答案
- **循序渐进**：从简单到复杂，从具体到抽象
- **多角度解释**：文字、公式、图示、例子结合
- **联系实际**：将抽象概念与实际应用联系
- **纠正误区**：指出常见的错误理解

### 3. 常见课程模块

#### 数学类
| 课程 | 常见问题类型 |
|------|--------------|
| 高等数学 | 极限、导数、积分、级数 |
| 线性代数 | 矩阵运算、特征值、线性空间 |
| 概率论 | 分布、期望、假设检验 |
| 离散数学 | 集合、图论、逻辑 |
| 数值分析 | 插值、数值积分、迭代法 |

#### 计算机类
| 课程 | 常见问题类型 |
|------|--------------|
| 数据结构 | 树、图、排序、查找 |
| 算法设计 | 动态规划、贪心、分治 |
| 操作系统 | 进程、内存、文件系统 |
| 计算机网络 | TCP/IP、HTTP、路由 |
| 机器学习 | 模型原理、损失函数、优化 |

#### 物理类
| 课程 | 常见问题类型 |
|------|--------------|
| 力学 | 牛顿定律、能量守恒 |
| 电磁学 | 电场、磁场、麦克斯韦方程 |
| 量子力学 | 波函数、薛定谔方程 |
| 热力学 | 熵、热力学定律 |

### 4. 解答结构

**概念解释**：
1. 直觉理解：用通俗语言解释
2. 形式定义：给出严格的数学定义
3. 关键性质：列出重要性质
4. 应用举例：给出具体例子
5. 常见误区：指出容易犯的错误

**计算方法**：
1. 问题类型识别
2. 方法选择依据
3. 步骤分解
4. 示例演示
5. 验证方法

**证明思路**：
1. 目标分析：要证明什么
2. 工具选择：用什么方法
3. 关键步骤：证明的核心思路
4. 完整过程：逐步展开

### 5. 常见误区纠正

| 领域 | 常见误区 | 正确理解 |
|------|----------|----------|
| 微积分 | 导数就是斜率 | 导数是变化率，斜率是几何解释 |
| 线性代数 | 矩阵乘法满足交换律 | AB ≠ BA（一般情况） |
| 概率 | 独立事件互斥 | 独立 ≠ 互斥 |
| 机器学习 | 过拟合 = 模型太复杂 | 过拟合 = 泛化能力差 |

## 输出模板

```markdown
## 问题解答

### 问题理解
[用自己的话复述用户的问题，确认理解正确]

### 核心概念

**[概念1]**
- 直觉理解：[通俗解释]
- 正式定义：[严格定义]
- 关键性质：[重要性质]

**[概念2]**
[同上]

### 详细解答

**方法一：[方法名称]**
[步骤化解答]

**方法二：[方法名称]**（如有）
[步骤化解答]

### 举例说明
[具体例子，帮助理解]

### 常见误区
[容易犯的错误和正确理解]

### 延伸阅读
[相关参考资料或进一步学习的方向]
```

## 注意事项

- 作业辅导时，引导用户思考，不直接给出完整答案
- 如果用户的问题涉及具体的代码实现，给出代码示例
- 对于抽象概念，尽量给出具体例子
- 如果用户的问题表述不清，先澄清再回答
- 鼓励用户提出进一步的问题
