# 公式推导模块

## 触发场景

- 用户需要推导数学公式
- 用户需要理解某个定理的证明
- 用户需要分析算法的时间/空间复杂度
- 用户对某个数学结论的来龙去脉感兴趣

## 工作流程

### 1. 理解推导目标

明确以下信息：
- **目标公式**：最终要得到的表达式
- **起点**：从什么条件或公式出发
- **领域背景**：物理、计算机、经济学等
- **详细程度**：完整推导还是关键步骤

如果用户描述模糊（如"推导一下反向传播"），确认具体目标：是推导权重更新公式，还是证明梯度的正确性？

### 2. 推导原则

- **每一步都要有依据**：说明使用了什么定理、公式或变换
- **符号一致性**：全文使用统一的符号系统，首步定义所有符号
- **逻辑完整**：不跳步，除非用户明确要求省略
- **可验证**：用户应该能独立验证每一步

### 3. 推导策略

#### 推导前：规划路径
在动手推导前，先规划推导路径：
1. 列出从起点到目标需要的关键中间结果
2. 识别可能的难点和分岔点
3. 选择最直接的路径

#### 推导中：常用技巧

**代数变换**：
- 因式分解、展开
- 分子分母同乘同除
- 换元法
- 配方法

**微积分**：
- 链式法则：$\frac{d}{dx}f(g(x)) = f'(g(x)) \cdot g'(x)$
- 分部积分：$\int u \, dv = uv - \int v \, du$
- 泰勒展开：$f(x) = \sum_{n=0}^{\infty} \frac{f^{(n)}(a)}{n!}(x-a)^n$
- 洛必达法则：$\lim_{x \to c} \frac{f(x)}{g(x)} = \lim_{x \to c} \frac{f'(x)}{g'(x)}$（当 $\frac{0}{0}$ 或 $\frac{\infty}{\infty}$）

**线性代数**：
- 矩阵分解（特征值、SVD）
- 正交化（Gram-Schmidt）
- 投影：$P = A(A^TA)^{-1}A^T$
- 矩阵求导：$\frac{\partial}{\partial \mathbf{x}} \mathbf{x}^T A \mathbf{x} = (A + A^T)\mathbf{x}$

**概率统计**：
- 贝叶斯定理：$P(A|B) = \frac{P(B|A)P(A)}{P(B)}$
- 全概率公式：$P(A) = \sum_i P(A|B_i)P(B_i)$
- 期望的线性性：$E[aX + bY] = aE[X] + bE[Y]$
- 方差：$\text{Var}(X) = E[X^2] - (E[X])^2$

### 4. 推导后：验证策略

**验证是推导的必要环节**，必须至少使用以下一种方法验证：

**代入特殊值**：
- 将变量代入 0、1、∞ 等特殊值，检查结果是否合理
- 例：推导出梯度公式后，令输入为 0，检查梯度是否为预期值

**量纲分析**：
- 检查等式两边的量纲是否一致
- 例：损失函数对权重的梯度，量纲应为 [损失]/[权重]

**极限情况**：
- 检查极端条件下的行为
- 例：当学习率→0 时，更新量应→0；当样本量→∞ 时，估计应收敛到真值

**数值验证**：
- 用具体数值代入推导结果，与直接计算比较
- 例：用 PyTorch 的自动微分验证手动推导的梯度

```python
import torch
# 数值验证梯度推导
w = torch.tensor([2.0], requires_grad=True)
x = torch.tensor([3.0])
y = (w * x) ** 2  # 假设推导的是这个函数的梯度
y.backward()
print(f"PyTorch 梯度: {w.grad.item()}")
print(f"手动推导结果: {2 * w.item() * x.item()**2}")  # 应一致
```

**对称性检查**：
- 如果公式对某些变量具有对称性，推导结果也应体现这种对称性

### 5. 输出格式

使用 LaTeX 数学环境：

**行内公式**：`$E = mc^2$`

**独立公式**：
```latex
$$\frac{\partial L}{\partial w} = \frac{1}{N} \sum_{i=1}^{N} (y_i - \hat{y}_i) \cdot x_i$$
```

**多行推导**（使用 aligned 环境，对齐等号）：
```latex
\begin{aligned}
L(w) &= \frac{1}{2N} \sum_{i=1}^{N} (y_i - wx_i)^2 \\
\frac{\partial L}{\partial w} &= \frac{1}{N} \sum_{i=1}^{N} (y_i - wx_i) \cdot (-x_i) \\
&= \frac{1}{N} \sum_{i=1}^{N} (wx_i - y_i) \cdot x_i
\end{aligned}
```

**条件推导**（使用 cases 环境）：
```latex
f(x) = \begin{cases}
x^2 & \text{if } x \geq 0 \\
-x^2 & \text{if } x < 0
\end{cases}
```

### 6. 常见公式类型

#### 机器学习/深度学习
- 损失函数推导（交叉熵、MSE、对比损失）
- 梯度计算（权重、偏置、嵌入层）
- 反向传播（链式法则的逐层应用）
- 注意力机制（QKV 计算、softmax 温度）
- 优化算法（动量、Adam 更新规则）

#### 数学分析
- 极限、连续性证明
- 级数收敛性判别
- 多元函数极值（Hessian 矩阵判别法）
- 不等式证明（Cauchy-Schwarz、Jensen）

#### 线性代数
- 矩阵求导（标量对矩阵、矩阵对矩阵）
- 特征值分解与 SVD
- 最小二乘解的正规方程
- PCA 的数学推导

#### 概率论
- 分布推导（从定义推导 PDF/PMF）
- 最大似然估计（MLE）
- 贝叶斯估计（MAP）
- EM 算法推导

## 注意事项

- 推导过程中，每一步都要说明依据（使用了什么定理或变换）
- 如果推导很长，先给出推导大纲（关键中间结果），再逐步展开
- 对于复杂的推导，可以分阶段进行，每个阶段有明确的中间结果
- 如果用户只需要关键步骤，可以省略常规代数运算，但要标注"省略常规代数运算"
- **必须验证推导结果**（至少一种验证方法）
- 当推导有多种路径时，简要说明选择的路径和理由

## 编码与平台兼容

- **Windows GBK 环境**：Python 数值验证脚本头部加 `sys.stdout = io.TextIOWrapper(sys.stdout.buffer, encoding='utf-8', errors='replace')` 或设置 `PYTHONIOENCODING=utf-8`
- **LaTeX 公式**：使用 XeLaTeX 编译，避免 pdfLaTeX 对 Unicode 数学符号的兼容问题
- 需要安装的包用 `scripts/dep_check.py numpy scipy` 预检

## 输出格式

默认 Markdown。公式推导推荐 LaTeX 输出（公式渲染质量最佳）。用户要求 DOCX 或 LaTeX 时，参考 `OUTPUT_DOCX.md` 或 `OUTPUT_LATEX.md`。
