# 文献综述模块

## 触发场景

- 用户要求梳理某个领域的研究现状
- 用户需要写文献综述或相关工作章节
- 用户想了解某主题的研究进展
- 用户需要分析多篇论文的共同点和差异

## 工作流程

### 1. 理解需求

明确以下信息：
- **研究主题**：核心研究问题或领域
- **文献范围**：用户提供的论文列表，或需要用户补充
- **综述目的**：课程作业、学位论文、期刊投稿等
- **组织方式**：按时间、按方法论、按主题、按应用场景

如果用户未提供具体文献，先基于领域知识构建综述框架，并标注"需补充具体文献"。不要编造论文标题或作者。

### 2. 文献搜索策略

当用户需要检索文献时，提供以下搜索建议：

**关键词策略**：
- 核心概念词 + 方法词 + 应用词（如："medical image segmentation + deep learning + CT"）
- 同义词扩展（如：CNN / convolutional network / convnet）
- 中英文双语检索

**推荐数据库**：
| 数据库 | 适用场景 | 特点 |
|--------|----------|------|
| Google Scholar | 综合检索 | 覆盖广，支持引用追踪 |
| Semantic Scholar | AI/CS 领域 | 支持语义搜索，有论文摘要 |
| arXiv | 预印本 | 最新研究，尚未正式发表 |
| Web of Science | 高质量期刊 | 支持引用分析 |
| CNKI / 万方 | 中文文献 | 国内期刊和学位论文 |

**文献筛选原则**：
- 优先选择高被引论文（奠基性工作）
- 近 3 年的论文反映最新进展
- 顶会/顶刊论文（CVPR、NeurIPS、TPAMI 等）保证质量
- 综述论文可快速了解领域全貌

### 3. 文献分析维度

对每篇论文提取：

| 维度 | 提取内容 | 用途 |
|------|----------|------|
| 研究问题 | 解决什么问题 | 归类到主题 |
| 方法论 | 采用什么方法 | 方法演进脉络 |
| 核心贡献 | 主要创新点 | 识别关键突破 |
| 局限性 | 研究的不足 | 引出研究空白 |
| 数据集/实验 | 使用的数据和评估方式 | 可比性分析 |
| 引用关系 | 谁引用了谁，基于谁的工作 | 演进脉络 |

### 4. 综述组织结构

**按主题组织**（推荐，适合大多数场景）：
```
## 2.1 [主题A]
[主题A下的研究脉络：问题定义→方法演进→当前SOTA→未解决问题]

## 2.2 [主题B]
[主题B下的研究脉络]

## 2.3 主题间的关系
[不同主题的联系和演进趋势]
```

**按时间组织**（适合梳理演进历程）：
```
## 2.1 早期研究（2015-2018）
[奠基性工作，定义问题和基本方法]

## 2.2 发展期（2019-2022）
[方法多样化，性能快速提升]

## 2.3 近期进展（2023-至今）
[新范式出现，跨领域融合]
```

**按方法论组织**（适合方法对比型综述）：
```
## 2.1 基于深度学习的方法
[端到端学习方法]

## 2.2 基于传统机器学习的方法
[手工特征+浅层分类器]

## 2.3 混合方法
[结合多种方法的优势]
```

### 5. 综述写作要点

**避免流水账**：不要"张三做了X，李四做了Y"式罗列。按主题综合，形成对话：
- ❌ "Wang et al. (2020) proposed method A. Li et al. (2021) proposed method B."
- ✅ "在解决[问题]方面，早期方法依赖[特征]（Wang et al., 2020），随后 Li et al. (2021) 通过[改进]突破了[瓶颈]。"

**体现批判性**：
- 指出各方法的优缺点和适用条件
- 比较不同方法的实验设置是否公平
- 识别实验结果中的矛盾和争议

**引出空白**：在综述末尾自然过渡到当前研究的不足，为自己的研究做铺垫。

**使用连接词**：
- 对比：However, In contrast, On the other hand, Unlike
- 递进：Moreover, Furthermore, In addition, Building upon
- 因果：Therefore, As a result, Consequently, This leads to
- 举例：For instance, Specifically, Notably, A case in point

### 6. 常见问题诊断

| 问题 | 表现 | 改进方法 |
|------|------|----------|
| 流水账式综述 | "A做了X，B做了Y，C做了Z" | 按主题组织，综合多篇论文的共同点和差异 |
| 缺少批判性分析 | 只描述不评价 | 每个小节末尾增加"局限性与讨论"段落 |
| 研究空白不明确 | 综述与自己的研究脱节 | 在综述末尾用"然而...仍存在以下不足"过渡 |
| 文献覆盖不全 | 遗漏重要文献 | 补充经典（高被引）和最新（近3年）文献 |
| 组织混乱 | 跳跃式叙述 | 选定一种组织方式，每个小节有明确的主题句 |
| 引用堆砌 | 一段话引用10+篇论文 | 每个论点引用2-3篇代表性工作即可 |

## 输出模板

```markdown
## 文献综述：[研究主题]

### 1. 引言
[研究背景（2-3句）→ 研究意义（1-2句）→ 本综述范围和组织方式（1句）]

### 2. 研究现状

#### 2.1 [主题/方向1]
[主题概述（1句）→ 方法演进脉络 → 关键突破 → 当前SOTA → 小结（1句）]

#### 2.2 [主题/方向2]
[同上结构]

#### 2.3 [主题/方向3]
[同上结构]

### 3. 研究方法对比

| 方法类别 | 代表工作 | 核心思想 | 优势 | 局限性 | 适用场景 |
|----------|----------|----------|------|--------|----------|
| [类别1] | [代表性论文] | [一句话概括] | [优点] | [缺点] | [何时使用] |
| [类别2] | ... | ... | ... | ... | ... |

### 4. 当前研究的局限性
[总结3-5个共性不足，每个不足用1-2句话说明，并指出可能的原因]

### 5. 未来研究方向
[基于上述局限性，提出3-5个可能的研究方向，每个方向说明为什么值得探索]

### 6. 本研究的定位（可选）
[说明本研究如何填补上述空白，与现有工作的区别]
```

## 注意事项

- 如果用户提供了论文 PDF 或链接，优先基于实际内容分析
- 如果用户没有提供具体文献，可以基于领域知识给出框架，但要注明需要补充具体文献
- 综述中引用的文献格式应与用户要求的引用风格一致（APA、IEEE、GB/T 7714 等）
- 中文综述注意使用规范的学术语言，避免口语化表达
- 综述长度根据目的调整：课程作业 3000-5000 字，学位论文章节 5000-10000 字，期刊投稿按目标要求

## 编码与平台兼容

- **Windows GBK 环境**：终端输出中文前，在脚本头部加 `sys.stdout = io.TextIOWrapper(sys.stdout.buffer, encoding='utf-8', errors='replace')` 或设置环境变量 `PYTHONIOENCODING=utf-8`
- **CSV/数据文件**：用 `scripts/quick_stats.py` 自动检测编码（utf-8 / gbk / latin-1）
- **LaTeX 编译**：始终使用 XeLaTeX（`compile.py` 自动检测），避免 pdfLaTeX 的中文 GBK 问题

## 工具脚本

分析脚本位于 `scripts/` 目录：
- `dep_check.py`：预检并安装缺失的 Python 依赖
- `quick_stats.py`：纯 stdlib 数据概览（无需 pandas）
- `analysis_report.py`：CSV → .tex 报告自动生成
- `encoding_safe.py`：Windows GBK 终端安全包装

## 输出格式

默认 Markdown。用户要求 DOCX 或 LaTeX 时，参考 `OUTPUT_DOCX.md` 或 `OUTPUT_LATEX.md`。
